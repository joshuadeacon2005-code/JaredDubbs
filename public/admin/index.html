    Your password

    Your admin password is stored as a Cloudflare Worker secret. To rotate it:

    1. Open the Cloudflare Worker settings page.
    2. Scroll to Variables and Secrets.
    3. Find ADMIN_PASSWORD, click Edit, set a new value, save.
    4. Wait ~30 seconds for the change to propagate, then come back here and log in with the new password.
    This logs out every device. Old admin sessions stop working the moment the password changes. There's no way to log out a single other browser without rotating — but rotating is fine and instant.

    Block out time on your calendar

    The website pulls availability from your Cliniko calendar. To stop the booking page from offering specific times:

    1. Open Cliniko and go to Calendar.
    2. Click and drag on an empty area to create an Unavailable block.
    3. Use that for lunch breaks, supervision, holidays, "no Mondays this month" — anything you don't want booked.
    4. For recurring blocks (e.g. lunch every weekday), tick Repeat when creating it.

    The website checks your Cliniko availability live, so changes show up on the next page load.

    Already-booked patient appointments are blocked automatically with a 30-minute buffer either side. You don't need to do anything for those.

    Change discovery call length or hours

    Length of the call (currently 15 minutes) — set on the appointment type in Cliniko:

    1. Cliniko → Settings → Appointment types.
    2. Open the discovery / introductory call type.
    3. Change Duration, save.

    Booking hours (currently 9am to 9pm Hong Kong time) — set as Cloudflare Worker variables:

    1. Open the Cloudflare Worker settings page.
    2. Under Variables and Secrets, find or add:
      • BOOKING_HOUR_START — first hour of the booking window (9 = 9:00am)
      • BOOKING_HOUR_END — first hour not offered (21 = last slot ends 9:00pm)
    3. Save. Changes are live in ~30 seconds.

    The website always offers slots in 30-minute steps within those hours, on any day Cliniko shows you as available.

    Pause online bookings

    If you need to stop the website from accepting bookings entirely (going on extended leave, system maintenance, etc.):

    1. Cliniko → Settings → Appointment types → open the discovery call type.
    2. Untick Show in online bookings, save.
    3. The website will show "Online booking is currently unavailable" and offer phone / WhatsApp instead.

    To restart bookings, re-tick the same checkbox.

    Where booking notification emails go

    When someone books a discovery call, an email is sent to hello@jareddubbs.com by default. To send to a different address:

    1. Open the Cloudflare Worker settings page.
    2. Set NOTIFY_EMAIL to the address you want.
    3. Save. Live in ~30 seconds.

    The email is sent via Resend. If you ever rotate that key, update RESEND_API_KEY the same way.

    If bookings stop working

    The booking page reads from Cliniko live. If the calendar shows "Online booking is currently unavailable" or no times appear, work through these in order:

    1. Check Cliniko itself. Log in to Cliniko and confirm: appointment type still exists, you're still listed as a practitioner, your business is still active.
    2. Check your free time. If your calendar is fully booked or fully blocked for the next 4 weeks, the website will show no slots. Add unavailable blocks back if you've removed something Cliniko was treating as available.
    3. API key rotated? If you regenerated the Cliniko API key, update CLINIKO_API_KEY in the Worker settings. The key ends in -au4 (or another shard) — that suffix matters; the website uses it to find the right Cliniko region.
    4. Renamed the appointment type? The website finds the discovery call by looking for "discovery", "introductory", "initial", or "free" in the name. If you've renamed it to something else (e.g. "Phone consult"), either rename it back or set CLINIKO_APPOINTMENT_TYPE_ID in Worker settings to lock it explicitly.
    5. Still broken? Send the developer the URL jareddubbs.com/api/cliniko/available-times response (open it in a browser tab) — the JSON shows exactly what's failing.

    Editing other things on the site

    Editable from this admin:

    • Blog posts — New Post and Existing Posts tabs.
    • Specific pieces of page text — Site Copy tab.

    Needs a developer to change:

    • The credentials bar at the top of the homepage (Master's, ADHD, Autism, LGBTQ+, SPACE).
    • Navigation menu items.
    • Footer links and contact details.
    • Page layouts and adding/removing sections.
    • Service pages beyond their intro paragraph (testimonials, FAQ, etc.).
    • Images and photos.

    If you need any of those changed, message the developer with what you'd like and where it is on the site.
